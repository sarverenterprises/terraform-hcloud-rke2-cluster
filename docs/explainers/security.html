Private Network Topology
🔒
Worker nodes — no public IPv4
assign_public_ip = false is the default for all worker pools. Nodes have a private IP on the Hetzner network only; they reach the internet via NAT.
🌐
LB uses private IP for targets
Control plane LB targets registered with use_private_ip = true — health check and proxy traffic stays on the private network, not the public interface.
🛡️
Cilium VXLAN over private net
Pod-to-pod traffic tunnels over UDP 8472 on the private subnet. The firewall restricts this port to cluster_subnet_cidr — the VXLAN fabric is invisible to the internet.
02
2Hardening
Encrypt & Isolate
etcd secrets are encrypted at rest, the join token is immutable after first apply, API credentials are scoped per component, and cloud-init logs are wiped post-boot.
Cluster-Level Hardening
✓
secrets-encryption: true — set in every control-plane cloud-init config. RKE2 writes an encryption config and rotates the AES-CBC key on each server node at boot.
✓
prevent_destroy lifecycle on the RKE2 join token resource. Rotating the token invalidates all node cloud-inits and would require full re-provisioning; the lock prevents accidental destruction.
✓
64-character random token generated by random_password with special = false (alphanumeric only, safe for shell embedding). Token is marked sensitive = true.
✓
cloud-init log truncation — boot scripts truncate /var/log/cloud-init-output.log and /var/log/cloud-init.log after RKE2 starts, removing token values that appear in rendered output.
✓
cni: none in the RKE2 config disables the default Canal CNI. Cilium is the only CNI present, preventing dual-stack networking confusion or Canal-to-Cilium policy bypass.
least-privilege tokens
Per-Component Hetzner API Tokens
🔑
CCM token — hcloud_ccm_token
Scoped to Cloud Controller Manager: node addressing, LB provisioning. Stored in a Kubernetes Secret, never in Helm chart values (which are stored in plaintext in cluster state).
🔑
CSI token — hcloud_csi_token
Scoped to CSI driver: volume create/attach/detach only. Separate token means a CSI compromise cannot manipulate LBs, servers, or networks.
🔑
Autoscaler token — hcloud_autoscaler_token
Scoped to Cluster Autoscaler: server create/delete for autoscaled pools. Isolated so autoscaler cannot reach storage or network APIs.
metadata API blocked
Cilium Cluster-Wide Network Policy
DENY · All Pods 🚫
Egress to 169.254.169.254/32
CiliumClusterwideNetworkPolicy: block-metadata-api — empty endpointSelector means every pod in every namespace is subject to this deny rule. Prevents workloads from reading Hetzner instance metadata, server tokens, or injected secrets.

Tailscale-Only
Entry Points
SSH and API access are gated behind the Tailscale CGNAT mesh. GitOps via Flux CD eliminates direct cluster access from CI pipelines entirely.
SSH Access Pattern
🔐
Tailscale node enrollment
When enable_tailscale_nodes = true, cloud-init installs Tailscale and joins with an ephemeral auth key tagged tag:k8s-node. Devices are removed from the tailnet automatically on shutdown.
🌐
SSH restricted to CGNAT range
Setting trusted_ssh_cidrs = ["100.64.0.0/10"] limits TCP/22 to Tailscale's CGNAT block. No public SSH access exists — reaching a node requires being on the tailnet first.
🎛️
Tailscale Kubernetes operator
Optional operator exposes Services onto the tailnet via ProxyGroup. OAuth credentials are pre-created in a Kubernetes Secret; the Helm chart values carry empty strings — secrets are never in Helm release manifests.
no direct kubectl in CI
GitOps — Flux CD Bootstrapping
🗝️
RSA-4096 deploy key generated
Terraform generates a TLS keypair at bootstrap. The private key is stored only in the flux-system Secret inside the cluster.
📖
Read-only GitHub deploy key
Auto-registration via flux_deploy_key_mode = "auto" posts the public key to GitHub with read_only: true. Flux can pull but never push to the GitOps repo.
🔄
No CI kubectl access required
Changes reach the cluster by merging to the Flux-tracked branch. CI pipelines never hold a kubeconfig — only Flux's source-controller clones and applies manifests.
kubeconfig handling
Kubeconfig — Local Only, Never Exposed
✓
Written to disk locally — null_resource.fetch_kubeconfig writes the file to .kube/<cluster_name>.yaml in the caller's working directory with chmod 600.
✓
Marked sensitive — the kubeconfig output is sensitive = true. Terraform will never print it in plan/apply output or expose it in the provider UI.
⚠
State backend encryption required — the kubeconfig is stored in Terraform state. The output description explicitly warns: "the state backend must use encryption."
✕
Never a plain output — the token output is gated by expose_rke2_token = false default. Token is always in state; plain-text exposure requires a conscious opt-in.
