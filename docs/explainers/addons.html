Cilium CNI — always deployed
🔷
eBPF networking — no flag needed
Cilium is unconditional. RKE2 is configured with cni: none so Canal is disabled; Cilium takes its place. VXLAN tunnel mode with MTU 1400 (1500 Hetzner NIC minus 50 bytes encapsulation overhead) prevents silent fragmentation on private networks.
🛡️
Metadata API blocked cluster-wide
A CiliumClusterwideNetworkPolicy denies all pod egress to 169.254.169.254, preventing workloads from reading Hetzner instance credentials or enumerating node metadata.
ℹ
kubeProxyReplacement = "false" (string) — full replacement breaks hcloud-ccm LoadBalancer reconciliation (rancher/rke2#4862). Must be a string, not a boolean.
ℹ
operator.replicas = 2 — a single Cilium Operator is a SPOF for IPAM; two replicas with leader election provide HA.
builds on Cilium

Longhorn — enable_longhorn
💾
Distributed block storage on /mnt/longhorn
Longhorn replicates volumes across worker nodes. Data lives on dedicated Hetzner block volumes mounted at /mnt/longhorn. Helm timeout is 600 s — it installs many DaemonSets and CRDs.
📊
Dynamic replica count
longhorn_default_replicas is computed as min(total_workers, 3) in the root module — a single-node cluster gets 1 replica; a 3+ node cluster gets 3. Avoids scheduling failures when the cluster is smaller than the default replica count.
creates two StorageClasses
RWO
longhorn-rwo — ReadWriteOnce (default) ext4, allowVolumeExpansion=true, reclaimPolicy=Delete, volumeBindingMode=Immediate
DEFAULT SC
RWX
longhorn-rwx — ReadWriteMany (builtin NFS) Only created when longhorn_rwx_mode = "builtin". NFS v4.1, softerr, timeo=600, retrans=5. Set "external" to use a separate NFS provisioner instead.
OPTIONAL
⚠
Monitoring dependency: kube-prometheus-stack requests longhorn-rwo for Prometheus (50 Gi) and Alertmanager (10 Gi). Enable Longhorn first or override storageSpec in monitoring values.
ℹ
longhorn_volume_size on a node pool definition pre-provisions a block volume of that size (GB) on each worker node during cloud-init.
autoscaler manages pool size
Cluster Autoscaler — enable_cluster_autoscaler
⚙️
HCLOUD_CLUSTER_CONFIG
Current Hetzner API (replaces deprecated HCLOUD_CLOUD_INIT). JSON base64-encoded into a Secret. Contains imagesForArch and per-pool cloudInit strings pre-rendered in the root module.
📋
Explicit node groups only
autoDiscovery=false. Only pools with scaling_mode = "autoscaled" appear in autoscalingGroups. Fixed pools are never touched by the autoscaler.
📐
least-waste expander
Prefers the node group that wastes the fewest resources. balance-similar-node-groups=true prevents imbalance when scaling up. skip-nodes-with-system-pods=false allows draining non-DaemonSet system pods.
⚠
Requires: hcloud_autoscaler_token (separate Hetzner token scoped to autoscaler actions). Tailscale node auth key injected into autoscaler cloud-inits when enable_tailscale_nodes=true.
ℹ
autoscaler_rbac_level = "upstream" (full SIG RBAC) or "minimal" (reduced scope). Default is "upstream" for out-of-the-box compatibility.
min_nodesper pool (default 1)

Flux CD — enable_flux
🔄
OCI Helm chart from ghcr.io
Flux v2 is installed via the fluxcd-community OCI chart (oci://ghcr.io/fluxcd-community/charts/flux2). No traditional Helm repo index — the chart is pulled directly from the registry.
🔑
RSA-4096 deploy key, auto-registered
A TLS keypair is generated and the public key is written to the flux-system Secret. When flux_deploy_key_mode = "auto" and a GitHub PAT is provided, Terraform registers the key on the target repo via the GitHub REST API — no manual copy-paste required.

Prometheus + Alertmanager + Grafana
Full kube-prometheus-stack ~> 67.0. Includes kube-state-metrics and node-exporter. Helm timeout 600 s — deploys CRDs, DaemonSets, and Deployments (3–8 min on a fresh cluster).

Tailscale — enable_tailscale_operator / enable_tailscale_nodes
Operator 🔗
enable_tailscale_operator
Deploys the Tailscale Kubernetes Operator (~> 1.76). Registers as <cluster_name>-operator on the tailnet. OAuth credentials read from operator-oauth Secret — never exposed in Helm values.
Nodes 🖥️
enable_tailscale_nodes
Injects tailscale_node_auth_key into worker and autoscaler cloud-inits so each node joins the tailnet at boot. Separate from the operator auth key — allows independent ACL tagging (tag:k8s-node vs tag:k8s-operator).
ℹ
Two independent flags: the operator exposes Services onto the tailnet; node enrollment gives SSH/API access over Tailscale. They can be combined or used separately.
ℹ
Autoscaler integration: when enable_tailscale_nodes=true, the node auth key is embedded in the HCLOUD_CLUSTER_CONFIG cloud-init so newly provisioned autoscaled nodes join the tailnet automatically.
